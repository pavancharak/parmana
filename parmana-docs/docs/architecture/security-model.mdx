## Signature Algorithms

Parmana supports pluggable digital signature providers.

### Implemented

| Algorithm | Status | Notes |
|-----------|--------|-------|
| Ed25519 | ✅ Default | High-performance classical digital signature |
| Dilithium3 (ML-DSA-65) | ✅ Supported | NIST-standardized post-quantum digital signature |

### Planned

| Algorithm | Status |
|-----------|--------|
| Dilithium5 (ML-DSA-87) | Planned |
| SPHINCS+ (SLH-DSA) | Planned |

Signature algorithms are selected using the runtime configuration.

```dotenv
SIGNATURE_PROVIDER=ed25519
```

or

```dotenv
SIGNATURE_PROVIDER=dilithium3
```

## Performance

Reference benchmark (50 sequential Receipt requests)

| Algorithm | Average | Median |
|-----------|---------|--------|
| Ed25519 | 2.87 ms | 1.31 ms |
| Dilithium3 (ML-DSA-65) | 5.59 ms | 1.64 ms |

The benchmark measures complete Receipt generation, including:

- policy execution
- hashing
- signing
- storage
- HTTP serialization

It is not an isolated cryptographic benchmark.
